## Что сделано на первом этапе

- Создана рабочая копия проекта в `parkmanager`.
- Авторизация стала возвращать фактического пользователя после входа, чтобы login log не записывал случайного admin при fallback-входе.
- Проверка пароля вынесена в отдельную функцию в `AuthProvider`.
- Нормализация логина вынесена в отдельную функцию, чтобы избежать расхождений по регистру и пробелам.
- Поиск по клиентам, авто, активным стоянкам, истории и должникам переведен на общий matcher.
- Добавлена единая нормализация:
  - ФИО и текста;
  - телефона, включая замену первой `8` на `7`;
  - госномера авто с учетом похожих латинских/кириллических символов.

## Что не трогалось в первом этапе

- Касса.
- Долги.
- Закрытие смен.
- Формулы отчетов.
- Reset/backup/import.
- Supabase schema/RLS.
- Данные live-базы.

## Следующий этап

Перед кассой и долгами нужно отдельно проверить сценарии:

- вход admin и manager;
- выход и повторный вход;
- поиск клиента по ФИО;
- поиск по основному и дополнительному телефону;
- поиск по госномеру в клиентах, парковке, истории и должниках;
- запись корректного пользователя в login log.

После этого можно переходить к кассе: наличные/безнал, carryOver, actualCash, закрытие смены и отчеты.
